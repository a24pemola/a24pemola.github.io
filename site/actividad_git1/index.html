    Practica Git y GitHub I

    Crear y configurar el repositorio

    1. Crea un repositorio en tu cuenta de GitHub llamado DEAW.
    2. Clónalo en tu ordenador para trabajar en local.

    README + Primeros pasos con Git

    1. Si aún no lo hiciste, crea un archivo README.md en tu repositorio local.
    2. Escribe un pequeño texto explicando para qué se usará el repositorio y en qué módulo.
    3. Haz un commit inicial con el mensaje: Comenzamos con los ejercicios de Git.
    4. Sube los cambios al repositorio remoto con un push.

    Ignorar archivos

    1. Crea un archivo llamado privado.txt en tu repositorio local.
    2. Crea una carpeta llamada privada en tu repositorio local.
    3. Configura Git para que ignore tanto el archivo como la carpeta.

    Añadir archivos y etiquetas

    1. Añade un archivo 1.txt al repositorio local.
    2. Crea un tag v0.1.
    3. Sube los cambios al repositorio remoto, incluyendo el tag.

    Configuración de la cuenta de GitHub

    1. Pon una foto de perfil en tu cuenta de GitHub.
    2. Activa la autenticación en dos pasos en GitHub.

    Uso social de GitHub

    1. Sigue en GitHub a dos compañeros de clase.
    2. Sigue los repos DEAW del resto de tus compañeros.
    3. Dale una estrella a los repos DEAW de tus compañeros.

    Añadir colaboradores

    1. Agrega a github.com/raul-profesor como colaborador en el repositorio DEAW.

    Trabajo con ramas

    1. Crea una rama v0.2.
    2. Cámbiate a la rama v0.2.
    3. Añade un archivo 2.txt en la rama v0.2.
    4. Sube la rama v0.2 al repositorio remoto.

    Merge directo

    1. Cámbiate a la rama master.
    2. Haz un merge de v0.2 en master.

    Merge con conflicto

    1. En la rama master, escribe Hola en 1.txt y haz commit.
    2. En la rama v0.2, escribe Adios en 1.txt y haz commit.
    3. Cámbiate a master y haz merge con v0.2 (esto generará un conflicto).

    Resolver conflictos

    1. Lista las ramas con y sin merge.
    2. Resuelve el conflicto anterior y haz un commit.

    Borrar ramas

    1. Crea un tag v0.2.
    2. Borra la rama v0.2.

    Listado de cambios

    1. Lista todos los commits, ramas y tags del repositorio.
